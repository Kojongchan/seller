## 🪟 대화창(워크스트림) 구조

> **원칙:** 프론트/백엔드로 창을 나누지 않습니다. Next.js는 풀스택 단일 저장소라
> 기능을 **수직 슬라이스(UI+API+데이터 한 번에)**로 만드는 게 효율적입니다.
> 그래서 "기능 개발"은 **Epic 단위**로 창을 나눕니다.

| # | 대화창 | 역할 | 주 산출물 |
|---|---|---|---|
| 1 | 🧭 **기획 (PM)** | 로드맵·기능 스펙·우선순위 관리. 다른 창이 참조하는 기준. | `PLAN.md` 갱신 |
| 2 | 🎨 **디자인 (UX/UI)** | 디자인 시스템, 화면 플로우, 컴포넌트 스펙 | `DESIGN.md` + 코드의 디자인 토큰/공통 컴포넌트 |
| 3 | ⚙️ **아키텍처/백엔드 기반** | 데이터 모델, API 설계, 외부 연동(네이버·쿠팡), 배치, 추후 DB/인증 | `ARCHITECTURE.md` 갱신 + 기반 코드 |
| 4 | 💻 **기능 개발 (Epic별)** | Epic 단위로 프론트+백 함께 구현. Epic마다 창 분리 가능 | 동작하는 기능 + PR |

**개발 순서:** E1 키워드/소싱 고도화 → E2 자동발주 → E3 상세페이지 자동화 → E4 자동 CS

## 📋 각 대화창 "시작 프롬프트" (복사해서 새 창에 붙여넣기)

### 1) 기획 창
```
이 저장소의 docs/README.md와 docs/PLAN.md를 먼저 읽어.
너는 이 프로젝트의 PM이야. PLAN.md의 로드맵/기능 스펙을 검토하고,
내가 논의하는 내용을 반영해 PLAN.md를 업데이트해줘. 코드는 건드리지 마.
```

### 2) 디자인 창
```
docs/README.md, docs/DESIGN.md, docs/PLAN.md를 읽어.
너는 UX/UI 디자이너야. 영상 레퍼런스(다크 테마 키워드 카드, 등급 뱃지, 검색량 추이 차트)를
참고해 디자인 시스템과 화면 플로우를 DESIGN.md에 정리하고, 필요한 공통 컴포넌트/토큰을 제안해줘.
```

### 3) 아키텍처 창
```
docs/README.md, docs/ARCHITECTURE.md, docs/PLAN.md를 읽어.
너는 백엔드/아키텍처 담당이야. E1을 구현하기 위한 데이터 모델·API 설계·외부 연동(네이버 데이터랩,
쿠팡)·배치 작업을 설계하고 ARCHITECTURE.md를 갱신해. 기반 코드부터 잡아줘.
```

### 4) 기능 개발 창 (예: E1)
```
docs/README.md, docs/PLAN.md(Epic E1), docs/ARCHITECTURE.md, docs/WORKFLOW.md를 읽어.
너는 개발자야. WORKFLOW.md의 브랜치→PR→프리뷰→머지 루프를 따라 Epic E1을 구현해.
작업 시작 전 PLAN.md의 E1 범위와 '열린 질문'을 확인하고, 모호하면 나에게 물어봐.
```
