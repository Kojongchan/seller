## 1. 비전 & 목표

**한 문장:** 과일 위탁판매 셀러의 반복 업무(소싱·발주·상세페이지·CS)를 자동화해,
**"무엇을 팔지 → 어떻게 팔지 → 운영"** 전 과정을 한 도구에서 끝낸다.

- 1차 목표: **나(운영자) 한 사람의 위탁판매를 자동화**하는 개인 도구
- 2차 목표(추후): 다른 셀러도 쓸 수 있는 서비스(SaaS)로 확장 가능한 구조 유지

## 3. 타겟 사용자 / 페르소나

- **주 사용자:** 과일 위탁판매를 하는 1인 셀러(=운영자 본인). 비개발자.
- **니즈:** "지금 뭘 올려야 잘 팔리나"를 빠르게 판단 + 주문/발주/CS 손품 줄이기.
- **사용 맥락:** 주로 데스크톱, 가끔 모바일로 트렌드 확인.

## 4. 제품 범위 — 5대 축

1. **상품 소싱 (키워드/트렌드 리서치)** ← 현재 위치, E1
2. **판매 계획** (무엇을 언제 얼마에 — 피크 예측 기반)
3. **자동 발주** (주문내역 → 도매처 발주서)
4. **상세페이지 자동화** (상품정보 → 이미지/문구)
5. **자동 CS** (문의/리뷰 응대)

### 🟢 E1 — 키워드/소싱 고도화 (지금)
영상의 "키워드 모음집/선정" 화면이 목표. **첫 산출물 = 키워드 분석기.**

> **🚦 개발 준비도(Go/No-Go) — 2026-06-18:**
> - ✅ **E1-1 (1a) 네이버만 분석기 = 구현·머지 완료 (2026-06-18, PR #5).** 임의 키워드 검색→2년 추이·피크·D-day·임시등급·세분류 비교·쿠팡 딥링크 라이브 반영.
> - 🔴 1b(경쟁수 등급) = 쿠팡 크롤링 조사(아키텍처 창) 선행 → 1a 위에 얹기.
> - 🔴 E1-2 인기검색어 TOP = 네이버 TOP500 API 조사 선행.
> - **1a 개발 단위:** ① `/analyze?q=` 라우트+메인 검색창 ② 트렌드 12→24개월 확장 ③ 1a 등급 산식+S~D 뱃지 ④ 4지표 카드(D-day)+2년 차트+세분류 비교.

**E1-1. 키워드 분석기 (먼저 구현)** — 과일/키워드 입력 → 한 화면에:
> 출시는 2단계 분리(확정): **1a = 네이버 데이터만으로 분석기 출시**(추이·피크·D-day·세분류 비교·임시 등급) → **1b = 쿠팡 경쟁상품수 크롤링 붙여 '경쟁 대비 기회' 등급 고도화.** 크롤링이 막혀도 1a는 독립적으로 가치 제공.
- [x] **2년 검색 추이 차트** (네이버 검색어트렌드, 좌=과거→우=최근) — ✅ 1a 완료 (**일별 선 차트**, `timeUnit=date`)
- [x] **시즌 피크 예측** — 예상 피크 검색량, 피크월, **D-day**, 작년/재작년 피크 비교 — ✅ 1a 완료 (예상 피크 **일자**까지 산출)
- [x] **등급(S/A/B/C/D)** — ✅ **1a 임시등급 완료**(`검색지수×0.6+시즌×0.4`, `경쟁 반영 전` 라벨). 1b 경쟁수 고도화는 미착수. **단계별 분리식**(2026-06-18 확정). 등급표(S~D)는 두 단계 공통, 입력만 다름:
  - **1a 임시등급(네이버만):** `점수 = 0.6 × 검색지수강도 + 0.4 × 시즌신호`
    - 검색지수강도 = 최근·피크 검색지수 수준(네이버 상대지수 0~100, 과일 간 비교는 동일 쿼리 묶음으로 정규화)
    - 시즌신호 = 피크 임박도(D-day) + 전년 대비 상승세
  - **1b 고도화(경쟁수 추가):** `기회점수 = 검색지수 ÷ 경쟁상품수(정규화)`를 핵심으로 두고 시즌으로 보정 → 동일 등급표로 환산
    - 경쟁상품수: **쿠팡 검색결과 크롤링(1순위)**, 실패 시 **검색량·시즌·급등세 신호(=1a식)로 자동 폴백**
- [x] **세분류 vs 대표키워드 비교** ("청송사과 vs 사과" — 세분류가 더 높으면 우량 신호) — ✅ 1a 완료 (자동추천 + 수동 A vs B)
- [x] 쿠팡 검색 바로가기 버튼 — ✅ 1a 완료 (`/np/search?q=` 딥링크)

#### E1-1 화면 스펙 (PM 확정, 2026-06-18)

**전제 — 분석 대상:** 분석기는 **검색하는 임의의 과일·키워드 전체**가 대상이다. 고정 16종이 아님.
- 네이버 검색어트렌드는 **키워드 기반(카테고리 불필요)** → 아무 키워드("샤인머스캣", "청송사과", "햇사과" 등) 분석 가능.
- `lib/fruits.ts`의 16종/샘플 시즌데이터는 **빠른-예시 칩 + 키 없을 때 폴백/데모용**일 뿐, 분석 범위를 한정하지 않는다.

**입력 방식 (확정):** **자유 입력 검색이 기본** + 16종(및 자주 보는 것)은 빠른-예시 칩.

**화면 IA (확정):** **분석 상세를 별도 화면으로 분리**하고, 메인은 진입점.
- `/` (메인 = 진입점): 큰 **검색창** + 빠른-예시 칩 + (E1-2) 추천/모음집 미리보기. 검색 제출 시 분석 상세로 이동.
- `/analyze?q=<키워드>` (분석 상세): 단일 키워드 심층 분석.

**분석 상세 화면 — 표시 항목 (위→아래 순):**
1. **헤더:** 키워드 · 데이터 출처 뱃지(`네이버 실데이터`/`샘플`) · **등급 뱃지(S~D)** — 1a에선 `경쟁 반영 전` 라벨 병기
2. **핵심 요약 4지표 카드:** ① 현재 검색지수 ② 예상 피크월 ③ 피크까지 **D-day** ④ 작년 대비 추세(↑/↓/→)
3. **2년 추이 차트:** 좌(과거)→우(최근), 성수기 구간 강조 + 이번 달 표시 + 작년/재작년 동월 비교
4. **시즌 피크 예측 블록:** 예상 피크월·피크지수, D-day, 작년·재작년 피크월 비교
5. **등급 근거:** 점수 산식 노출 — 1a식(검색지수×0.6+시즌×0.4) / 1b식(검색지수÷경쟁상품수+시즌). 경쟁상품수는 1b에서, 없으면 `집계 전`
6. **세분류 비교 (확정: 자동 추천 + 수동 비교 둘다):** 입력 키워드의 대표 세분류(지역/품종) 2~3개를 **자동 추천**해 대표키워드와 막대 비교 + 사용자가 직접 `키워드 A vs B` 수동 비교
7. **액션:** **쿠팡 검색 바로가기**(딥링크) · (E1-2) 모음집에 추가 / 알람 설정

**상태 / 폴백:** 로딩 · 빈 검색 · 트렌드 없음(지수 0) · 네이버 실패→샘플 폴백 · 등급 1a(임시)/1b(경쟁 반영). 1a 출시 시 경쟁상품수·일부 세분류는 placeholder 허용.

**디자인 창 핸드오프:** 등급 뱃지 색/스케일, 4지표 카드, 2년 차트(피크·동월 강조) 시각 규격 → `DESIGN.md`.
