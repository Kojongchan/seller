---

## 1. 비전 & 목표

**한 문장:** 과일 위탁판매 셀러의 반복 업무(소싱·발주·상세페이지·CS)를 자동화해,
**"무엇을 팔지 → 어떻게 팔지 → 운영"** 전 과정을 한 도구에서 끝낸다.

- 1차 목표: **나(운영자) 한 사람의 위탁판매를 자동화**하는 개인 도구
- 2차 목표(추후): 다른 셀러도 쓸 수 있는 서비스(SaaS)로 확장 가능한 구조 유지

---

## 2. 핵심 결정 (2026-06-18 확정)

| 항목 | 결정 | 함의 |
|---|---|---|
| 제품 성격 | **내 판매 자동화 우선** (단일 사용자) | 초기엔 **로그인/다중사용자/결제 불필요** → 빠른 구현. 단, 데이터 구조는 추후 멀티테넌트 확장 가능하게. |
| 판매 채널 | **쿠팡 우선**, 추후 확장(스마트스토어 등) | 발주/상세페이지/CS 자동화는 쿠팡 기준 우선 설계 |
| 키워드 리서치 소스 | **네이버 데이터랩** (검색어트렌드 + 쇼핑인사이트) | 이미 연동 완료. 쿠팡엔 동등한 트렌드 API가 약해서 리서치는 네이버로. |
| 1순위 개발 | **E1 키워드/소싱 고도화** | 이미 있는 기능을 키우는 것이라 리스크 최저 |

---

## 3. 타겟 사용자 / 페르소나

- **주 사용자:** 과일 위탁판매를 하는 1인 셀러(=운영자 본인). 비개발자.
- **니즈:** "지금 뭘 올려야 잘 팔리나"를 빠르게 판단 + 주문/발주/CS 손품 줄이기.
- **사용 맥락:** 주로 데스크톱, 가끔 모바일로 트렌드 확인.

---

## 4. 제품 범위 — 5대 축

1. **상품 소싱 (키워드/트렌드 리서치)** ← 현재 위치, E1
2. **판매 계획** (무엇을 언제 얼마에 — 피크 예측 기반)
3. **자동 발주** (주문내역 → 도매처 발주서)
4. **상세페이지 자동화** (상품정보 → 이미지/문구)
5. **자동 CS** (문의/리뷰 응대)

---

## 5. 로드맵 — Epic

각 Epic은 **수직 슬라이스**(UI+API+데이터)로 구현하고, 끝나면 라이브 배포.

### 🟢 E1 — 키워드/소싱 고도화 (지금)
영상의 "키워드 모음집/선정" 화면을 목표로 현재 트렌드 기능을 발전시킨다.
- [ ] **과일 인기검색어 TOP 리스트** (네이버 쇼핑인사이트 분야별 인기검색어)
- [ ] **키워드 등급(S/A/B…)** 산정 로직 (검색량·시즌성·경쟁도 조합)
- [ ] **시즌 피크 예측** — 예상 피크 검색량, 피크월, D-day (과거 2년 데이터 기반)
- [ ] **세분류/롱테일 키워드 발굴** (예: 사과 > 청송사과 > 세척사과 / 자동완성·연관검색어)
- [ ] **실데이터 기반 "이번 달 추천"** (현재 샘플 → 실데이터 전환)
- [ ] UX 정리: 차트 월 정렬(달력순 옵션), 작년/재작년 비교, 쿠팡 검색 바로가기
- **열린 질문:**
  - 쿠팡 인기검색어 TOP500이 네이버 API로 어디까지 제공되는지 확인 필요 (미제공 시 대안)
  - 등급 산정에 쓸 "경쟁도/상품수"를 어디서 가져올지 (쿠팡 검색결과? 별도)

### ⚪ E2 — 자동 발주 시스템
- [ ] 주문내역 업로드(엑셀/CSV) 파싱
- [ ] 상품 ↔ 도매처 매핑 테이블 (저장 필요 → 이때 경량 DB 도입 검토)
- [ ] 도매처별 발주서 자동 생성/다운로드
- **열린 질문:** 쿠팡 주문내역 양식, 도매처 발주 포맷 표준화 범위

### ⚪ E3 — 상세페이지 자동화
- [ ] 상품 정보 입력 → 상세페이지 문구/구성 자동 생성 (AI 카피 + 템플릿)
- [ ] 이미지 템플릿/조합
- **열린 질문:** 어느 정도 자동 vs 반자동(검수)

### ⚪ E4 — 자동 CS 관리
- [ ] 문의/리뷰 분류 + 응대 템플릿 추천/자동화
- **열린 질문:** 쿠팡 CS 채널 접근 방법(수동/연동)

## 6. 성공 지표 (초기)

- E1: "오늘 뭘 올릴지"를 1분 안에 결정 가능 (추천 + 등급 + 피크 한 화면)
- E2: 주문 100건 발주서 생성 시간 = 수작업 대비 90%↓
- 전체: 운영자의 주간 반복작업 시간 측정 → 단계별 절감
